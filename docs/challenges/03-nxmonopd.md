# Challenge 3 — Convert to NXmonopd and Identify Missing Information

## Goal
Convert the provided experimental dataset into a NeXus file that follows the **NXmonopd application definition**. Include an **NXnote** within the same NXentry that documents missing experimental or calibration information and any limitations of the application definition for this dataset. The final file must be uploaded to NOMAD.

## What you get
- A directory containing raw experimental data files  
- A master metadata file with scan parameters  
- A minimal ELN YAML file with partial metadata  
- A mapping template (in JSON format) showing how fields can be mapped to NeXus structure (incomplete and must be adapted)
